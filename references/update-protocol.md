# DR-ZHOU Update Protocol

Use this file when the user asks to update DR-ZHOU with new paper examples, checklists, writing preferences, reviewer experiences, or student-paper problems.

## Update Steps

1. Identify the material type:
   - writing rule
   - before/after example
   - preferred phrase
   - forbidden phrase
   - section pattern
   - figure/table standard
   - reviewer-response habit
   - domain-specific technical preference
2. Distill the material into reusable rules.
3. Add the rule to the most relevant reference file.
4. If the material changes triggering or workflow behavior, update `SKILL.md`.
5. Append a dated entry to `references/update-log.md`.
6. Run skill validation after structural edits.

## Storage Principles

- Prefer distilled rules over raw archives.
- Keep examples short and representative.
- Do not duplicate the same rule across multiple files.
- If a new rule conflicts with an old rule, keep the newer user-provided rule and record the conflict.
- If the user provides a full paper, extract style patterns and section logic rather than copying the full paper into the skill.

## Suggested User Prompt for Updates

```text
用这条经验更新 DR-ZHOU：<粘贴经验、例子或修改偏好>
```

or

```text
把这个学生常见问题加入 DR-ZHOU，以后遇到类似写法要提醒：<问题>
```
